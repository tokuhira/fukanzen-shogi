# 不完全将棋 docs 索引

このディレクトリの dashboard。真実は各文書にある——ここは**索引**であり、内容の要約や重複は持たない。このアークの締めで**ディレクトリを階層化**した: 現行の権威（`design/`）と歴史・完了記録（`archive/`）を物理的に分け、生きた設計が履歴に埋もれない形にした。

**使い方**: 現行の挙動・ルールを知りたければ `design/` と本索引の A だけを読めば足りる。「なぜ今の形になったか」を追うときだけ `archive/`（B・C・D）を辿る。

**ディレクトリ構成**:

```
docs/
  README.md                          ← 本索引
  design/                            ← A. 生きた設計文書（現行の権威）
  不完全将棋_バックログ_伏線と未決.md   ← 開いている畝の索引（現在進行形・トップに置く）
  archive/                           ← 完了記録・歴史（掘り起こすときだけ）
    board-split_総括_第零段から第三段b-3.md   ← board.js 分割アークの総括＋解決済み台帳
    implementation/                  ← C. 完了した実装指示書
      board-split/                   ←   board.js 分割 第〇〜三段b-3（10 本）
      （淀川・記録係・その他は implementation/ 直下または小分類）
    rule-history/                    ← B-1. ルール仕様 v0.1〜v0.5
    sengoku-musou/                   ← B-2. 実装変更指示 v0.3〜v0.5（戦国無双系）
```

※ディレクトリ名は英語、ファイル名は日本語のまま。
