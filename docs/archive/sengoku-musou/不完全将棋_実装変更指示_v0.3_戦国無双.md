# 不完全将棋 実装変更指示 v0.3 — 玉の戦国無双特則

> 対象実行者: Claude Code
> 前提: 第一段階（Phase 1）が実装済みであること（`engine/` のルールエンジン、`cli/` の検証 CLI、回帰テスト群が存在し、全テストが通る状態）。
> 関連文書: `不完全将棋_ルール仕様_v0.3.md`（新規則の正本）、`不完全将棋_実装指示書_第一段階.md`（Phase 1 の全体設計）。
> 性格: 本書はゼロからの実装指示ではなく、**既存の動作する実装に対する一点の規則変更**の指示である。変更は外科的で、影響範囲は衝突解決の一箇所と、それに付随するテストに限られる。

---

## 0. なぜ変えるのか（背景）

第一段階の実機テストで、ゲームバランスの問題が観測された。中核は **王手の価値が一方的に高すぎる**こと。走り駒の王手には合駒が効かず（経路非干渉）、玉は利きに覆われると確定的に取られるため、王手された側は回避一択に追い込まれる。とりわけ、**後ろ盾のない大駒・カナ駒を玉の至近へ打ち込む「鉄砲玉の暗殺」**が高勝率の必勝手段と化し、侍同士の合戦ではなく忍者の暗殺ゲームになっていた。これに対抗するには駒を取られないよう守るしかなく、積極策が罰され消極策が最適化される——方針3（膠着打開への報酬）と正反対の力学が働いていた。

本変更は、この暗殺の最も安直な形を、**最小の規則変更で**罰する。

---

## 1. 変更の本体（戦国無双特則）

仕様書 v0.3 の **4.7** を実装する。規則は次の一条に集約される。

> 相互移動（スワップ＝2枚の駒が互いに相手の旧位置を移動先とする正面衝突。既存の 4.4）の当事者の一方が玉である場合、4.4 の相討ちを適用しない。**玉は取られず、相手の駒のみを取って自分の持ち駒とし、玉は相手の旧位置へ進む。**

### 1.1 正確な動作

スワップ `As`（玉、`Fking → Tking`）と `Ag`（相手の駒、`Fenemy → Tenemy`）について、`Tking == Fenemy` かつ `Tenemy == Fking`（互いの旧位置を目指す正面衝突）が成立し、かつ一方が玉のとき:

- 玉は **相討ちにならない**（盤上から取り除かれず、相手の持ち駒にもならない）。
- 玉は相手の駒を取り、その駒は **玉側の持ち駒**となる（取った駒は基本種に戻す。通常の取得と同じ）。
- 玉は相手の駒の旧位置 `Tking`（＝ `Fenemy`）へ進む。
- 相手の駒の移動（`Fenemy → Tenemy == Fking`）は成立しない（その駒はもう取られているため、玉の旧位置へは到達しない。玉の旧位置 `Fking` は空になる）。

### 1.2 合法性との関係（重要・既存ロジックで自動的に担保される）

**この特則は玉の侵入禁止に優先しない。** 玉の移動先 `Tking`（相手の駒の旧位置）が着手開始時点で相手の利きにあるマスなら、その玉の移動は**そもそも合法手生成に現れない**（既存の `legal_actions` が伝統的将棋の合法手として弾く）。したがって:

- 玉がこの一方的取得を発動できるのは、`Tking` が安全なマス＝**相手の駒に後ろ盾（味方の利き）が無い**ときに限られる。これがまさに「鉄砲玉だけ斬り返せる」という意図した挙動になる。
- **この絞り込みは新たに実装する必要がない。** 合法性判定（着手開始時点で玉が利きに侵入できない）が既に正しく実装されていれば、スワップが成立する時点で `Tking` は必ず安全であることが保証される。本変更で触るのは**解決（resolve）の挙動のみ**であり、合法性判定は一切変更しない。

> 実装メモ: もし `resolve` が「合法手のみが渡される」前提で書かれているなら、`Tking` の安全性を resolve 内で再検査する必要はない（合法性で担保済み）。ただし防御的に `debug_assert!` で「玉の移動先が着手開始時点で相手の利きにないこと」を確認するのは歓迎。

### 1.3 変更しないこと（誤って広げないための明示）

戦国無双が救うのは **スワップ（入れ違い）の一点のみ**。以下は **従来どおり**で、変更してはならない:

- **玉が留まったまま、隣接する相手の駒が玉のマスへ移動してくる（取得）** → 玉は救われず死ぬ（5.2 玉の死の主経路）。玉が斬り返せるのは自ら動いて入れ違うときだけ。じっと留まっていれば討たれる。
- **相手の駒が玉の旧位置以外へ逃げた場合** → スワップではない。玉から見て「逃げた駒（4.2）」であり、取得は発生せず玉は空いたマスへ進むだけ。
- **玉以外の駒同士のスワップ** → 従来どおり相討ち（4.4）。特則は玉が当事者のときだけ。
- 取得（4.1）・逃げた駒（4.2）・同一マスの相討ち（4.3）・相討ちの交換（4.5）・経路非干渉（4.6）→ すべて不変。

## 2. 実装箇所

変更は衝突解決ロジック（おそらく `engine/src/resolve.rs` のスワップ判定／4.4 相当の分岐）に局所化されるはずである。

1. スワップ（相互移動）を検出した分岐で、**当事者の一方が玉か**を判定する分岐を追加する。
2. 一方が玉なら、相討ちではなく「玉が相手駒を取得して `Tking` へ進み、玉自身は生存」という解決を行う。取った駒は玉側の持ち駒へ（基本種に戻す）。玉の旧位置は空にする。
3. 玉以外同士のスワップ、および玉の死の主経路（取得）は一切変えない。

> 「同一マスの当事者は最大2枚」「干渉は最終マスのみ」という既存の不変条件は保たれる。本変更はスワップ解決の**結果**を変えるだけで、衝突の**検出**構造は変えない。

## 3. テストの変更

### 3.1 追加するテスト

- **戦国無双・基本**: 玉に隣接する後ろ盾のない相手の駒（例: 単騎で打ち込まれた銀）と玉が入れ違うスワップで、**玉が生存し、相手の駒を取って持ち駒に加え、玉が相手駒の旧位置へ進む**ことを確認。相手の駒は玉の旧位置へ到達しない。
- **戦国無双が発動しない（後ろ盾あり）**: 相手の駒が別の味方駒の利きで支えられている場合、その駒の旧位置（玉の移動先）が相手の利きでカバーされているため、**玉のその移動は合法手に現れない**ことを確認（`legal_actions` に当該の玉の手が含まれない）。すなわち戦国無双は支えのある駒には働かない。
- **留まり取得では救われない（主経路は不変）**: 玉が留まったまま、隣接する相手の駒が玉のマスへ移動 → **玉は取られて負け**（戦国無双はスワップ限定であり、この取得を救わないこと）を確認。
- **逃げた駒との区別**: 相手の駒が玉の旧位置以外へ逃げたスワップでない局面で、玉が相手駒の旧位置へ進む → 取得は発生せず空きマスを占めるだけ（4.2）であることを確認。

### 3.2 見直すテスト

- **「両玉同時の相討ち → 引き分け」テスト（Phase 1 のテスト17 相当）**: このシナリオ（両玉が同一マスへ移動して相討ち）は、玉の侵入禁止により**そもそも発生し得ない**（互いに相手の利きに侵入できないため、両玉が同一マスへ移動することも、玉が玉を取りに行くこともできない）。仕様書 v0.3 の 5.4 でこれを明記した。当該テストが「発生し得ない状況を無理に構成して引き分けを確認している」なら、削除するか、**実際に起こり得る引き分け経路**＝「両者が同一ターンに相手玉のマスへ駒を進める取得が成立し、両玉が同時に取られる」シナリオに置き換える。

### 3.3 回帰

- 既存テストのうち、玉が絡まない衝突（取得・逃げ・空きマス相討ち・玉以外のスワップ・経路非干渉・打ちの相討ち・成り解除）は**すべてそのまま通る**こと。本変更が玉以外の挙動に影響していないことを回帰で確認する。

---

## 4. やらないこと（非ゴール）

- **大駒の王手・打ち込みの強さを弱めないこと。** 龍・馬・飛・角の遠隔王手と、支えを伴う打ち込みは、積極攻勢への報酬として**意図的に強いまま残す**（仕様書 v0.3 第7章）。戦国無双が抑止するのは「後ろ盾のない鉄砲玉の一撃暗殺」のみ。これを超えて王手全般や打ち込み全般を抑制する変更を加えてはならない。
- 千日手・連続王手・打ち歩詰め・入玉宣言などの未確定事項は、本変更の対象外。従来どおり暫定処理＋印のまま。
- 通信・Wasm・CPU 等の後続段階には触れない。

---

## 5. 完了基準

1. 仕様書 v0.3 の 4.7 が実装され、3.1 の追加テストが全て通る。
2. 既存の回帰テスト（玉が絡まない衝突）が全て通る。
3. 「両玉同時相討ち」テストが、発生し得ない構成から、実際に起こり得る引き分け経路（両玉同時取得）へ置き換えられている、または削除されている。
4. 合法性判定（玉の侵入禁止）は一切変更されておらず、戦国無双の絞り込み（鉄砲玉限定）がその既存ロジックによって自動的に担保されていることが、テストで確認されている。
5. 大駒の強さに関わる挙動が弱められていない。

## 6. 検証後の所見記録（任意だが推奨）

本変更は実機テストの所見に基づくバランス調整である。変更後も再びテストプレイし、次を観察して所見として記録すること（修正はせず所見に留める。次の規則判断は人間が行う）:

- 後ろ盾のない打ち込み暗殺が実際に無力化されたか。
- 残した大駒の遠隔王手・支え付き打ち込みが、健全な積極性として機能しているか、それとも依然として消極策を強制する過剰な圧力になっているか（後者なら、走り駒の追走に対する玉の逃走特則など、次版での追加手当ての候補になる）。
- 一つだけ変えて測る原則を守り、複数の変更を同時に入れないこと。
