# 不完全将棋 実装指示書 — Web 盤の操作可能化（ホットシート・合法手提示）

> 対象実行者: Claude Code
> 前提: `web/` は Wasm 化した `engine` を読み込み、棋譜（初期局面＋着手列）をエンジン計算で再生できる（水墨様式）。Wasm 境界は SFEN/USI の文字列。`engine-wasm` クレートが `engine`（純粋・無改変）をラップ。ルールは v0.5。
> 関連文書: `不完全将棋_実装指示書_Web盤Wasm組み込み.md`、`不完全将棋_実装指示書_第二段階_TUI検証卓.md`（操作モデルの先行例）、現行ルール仕様 v0.5。
> 性格: Web 盤を「鑑賞するもの」から「触れるもの」へ。一人で両者を操作するホットシート（秘匿・通信なし）。TUI 検証卓の操作モデルを Web のマウス操作へ移植する。**鑑賞（棋譜再生）は操作の中に溶け込ませる**——指せば手が積まれ、前後に辿れば再生になる。エンジンは無改変。

---

## 0. 目的と範囲

- **作るもの**: Web 盤で、一人がマウスで先手・後手の両方の着手を組み立て、同時解決して局面を進められるようにする（ホットシート）。駒を選ぶと合法手を盤上に提示する（TUI 検証卓の目玉機能の Web 版）。指した手は棋譜として積まれ、前後に辿れる（鑑賞＝操作の中に統合）。
- **やらないこと（非ゴール、後の殻）**:
  - 秘匿・通信対戦（WebSocket/WebRTC）。本段階は単一画面・一人操作・両方可視。
  - CPU 対戦。
  - 水墨様式そのものの作り直し（既存の見た目を踏襲。新たに加わる動的要素＝合法手提示だけ、水墨に馴染む形で足す）。
- **一つ変えて測る**: 本段階で加わる本質は「操作」と「合法手提示」。Wasm 境界には `legal_actions` を足すが、再生（解決関数）は既存のまま流用。

---

## 1. Wasm 境界の追加：`legal_actions`

操作には、ある局面・ある陣営の合法手の列挙が要る。`engine-wasm` に境界関数を一つ足す（再生で使った解決関数はそのまま）。

- **合法手関数**: 入力 = 局面の SFEN、陣営（先手/後手）。出力 = その陣営の合法手の **USI のリスト**（文字列の配列、または改行区切り等の文字列）。内部で `engine::legal_actions` を呼ぶ。
- 境界は引き続き **SFEN/USI の文字列**で薄く保つ（内部型を JS に晒さない）。`engine` 本体は**無改変**。
- これにより JS は「この局面のこの陣営の合法手」を問い合わせ、駒選択時にその駒に関する手だけを抽出して盤上に提示できる（§3）。

---

## 2. 操作モデル（TUI の移植：選ぶ→提示→仮確定→解決）

TUI 検証卓の「駒を選ぶ→合法手提示→移動先で仮確定→成り選択→先後を組んで解決」を、マウス操作へ移植する。一人が両者を順に組む（両方可視。秘匿は後の殻）。

### 2.1 着手の組み立て（移動）

1. 盤上の自陣の駒を**クリックで選択**。
2. 選択した瞬間、その駒の**合法手を盤上に提示**（§3。`legal_actions` から当該駒の手を抽出）。
3. 提示されたマスをクリックで**仮確定**。
   - 成り可能な手は**成る／成らず**を問う（小さな選択 UI）。成れない駒・位置では問わない。行き所のない駒の不成はそもそも合法手に出ない（エンジンが除く）。
4. 仮確定した着手をテキストで表示（例: 先手「7六歩」）。

### 2.2 着手の組み立て（打ち）

1. 駒台の持ち駒を**クリックで選択**。
2. 打てる合法マスを**提示**（二歩・行き所のない駒・打ち歩詰めはエンジンが除くので、合法な着地のみ提示）。
3. 着地マスをクリックで仮確定（打ちは成り選択なし）。

### 2.3 先後を組んで解決

- 先手の手を仮確定し、続いて後手の手を仮確定（**両方可視**でよい＝検証的ホットシート）。
- 両者そろったら**解決**（ボタン等）で、既存の Wasm 解決関数に現局面 SFEN＋両 USI を渡し、解決後局面を得て描画。
- 解決前は仮確定を**取り消し**できる（選び直し）。
- 解決後、発生事象（取得・相討ち・スワップ・戦国無双・両玉スワップ引き分け・玉の死等）をテキスト表示（再生時と同じ表示を流用）。終局なら終局表示。

---

## 3. 合法手の提示（水墨を壊さない最小限の強調）

TUI の目玉機能「駒を選ぶと合法手がインタラクティブに提示される」を Web へ。ただし**水墨様式の静けさを壊さない**抑制された表現にする。

- 駒（盤上・駒台）を選択した瞬間、`legal_actions` の結果から**その駒に関する手だけ**を抽出し、移動先（打ち先）マスを盤上で**控えめに**示す。
- **表現は最小限の強調**（派手なハイライトは侘びを壊す）。候補例: 墨の淡い陰、細い枠、小さな点（路傍の標のような小さな印）。色を増やさず、墨の濃淡で示すのが水墨に忠実。**具体の見せ方は実装者がまず最小で作り、後でユーザーが美意識に沿って微調整できる余地を残す**。
- これにより、v0.5 のルールが対話的に可視化される: 玉を選ぶと相手の利きへ侵入する手は提示されない（侵入禁止）、王手時は回避手のみ提示、戦国無双が効かない（後ろ盾のある駒へ向かう玉の手が出ない）等が、選ぶだけで見える。
- 提示は「合法な行き先」を示すだけでよい。解決結果の予測（取得が起きるか等）まで盤上に描かない（検証者は合法手集合を見たいのであって、結果は解決して確かめる。これも“抜き”に通じる）。

---

## 4. 鑑賞を操作に溶け込ませる（棋譜という一本の背骨）

別々の「再生モード／操作モード」を作らない。**操作が主で、鑑賞はその中に含まれる**。

- **棋譜が背骨**: 対局は「初期局面＋着手列」の棋譜として保持（再生で使った構造そのもの）。指して解決するたびに、その (先手 USI, 後手 USI) が棋譜の末尾に**積まれる**。
- **前後に辿る＝再生**: 棋譜の任意の地点へ前後移動できる（既存の前後操作を流用）。過去の局面へ戻れば、そこまでの再生になる。戻った地点から指し直せば、そこから棋譜が分岐・更新される（末尾を置き換えて以降を再計算、という素直な扱いでよい）。
- **用意した棋譜の読み込み＝鑑賞**: 既存の「用意した棋譜」を読み込めば、それは操作系の中で「誰かが指した手の列」として前後に辿れる。鑑賞は、操作系における「既に積まれた棋譜を辿る」状態にすぎない。
- **局面の真実は SFEN、進行の真実は着手列**: 各局面は初期局面＋着手列からエンジンが再計算できる（解決後局面を別途持たない＝Wasm 組み込みで確立した方針を維持）。前後移動は、その地点までを再計算して描く。

> これにより、Web 盤は「鑑賞用ビューア」と「操作卓」の二顔を持たず、**着手列を前後に辿りつつ末尾で指し継げる一つの卓**になる。TUI で地続きだった再生と操作の関係が、Web でも一本に通る。

---

## 5. 画面（既存の水墨様式を踏襲し、操作要素を最小限足す）

- **盤面・駒台・座標・手数・解決結果テキスト**: 既存の水墨様式を踏襲。
- **加わる要素**:
  - 選択中の駒と、その合法手の控えめな提示（§3）。
  - 先手・後手の仮確定した着手のテキスト表示。
  - 解決ボタン、仮確定の取り消し。
  - 前後操作（既存）に加え、末尾で指し継げること。
- **入力フェーズの表示**: いま先手を組んでいるのか、後手か、解決待ちか、過去局面を閲覧中か、を控えめに示す。
- 凝った演出はしない（水墨の静けさを保つ。新規の動的要素は合法手提示と仮着手表示に限る）。

---

## 6. 完了基準

1. `engine-wasm` に `legal_actions` の文字列境界（SFEN＋陣営 → USI リスト）が加わる。`engine` 本体は無改変。
2. マウスで、移動・打ち・成りを含む着手を、先手・後手の両方について組み立てられる。
3. 駒を選ぶと、その駒の合法手が**水墨を壊さない控えめな表現**で盤上に提示される。v0.5 の合法性（玉の侵入禁止・王手回避絞り込み・戦国無双の非発動等）が提示の有無として見て取れる。
4. 両者の手を組んで解決でき、解決後局面と発生事象が表示される。終局（確定的詰み・玉の死・両玉スワップ引き分け）まで指せる。
5. 指した手が棋譜に積まれ、前後に辿れ、過去局面から指し継げる（鑑賞と操作が一本の棋譜に統合）。用意した棋譜を読み込めば同じ操作系で辿れる。
6. 既存の水墨様式・再生の所作は保たれる（新規の動的要素は合法手提示と仮着手表示のみ）。
7. 秘匿・通信・CPU は含まない（範囲を守る）。ローカル（簡易 HTTP サーバ）と Cloudflare（カスタムドメイン）で動く。

---

## 7. 次の殻（本段階では作らない・見通し）

- **秘匿対戦**: `protocol` の純粋論理を Wasm 化し、WebSocket/WebRTC の殻で繋ぐ。ホットシートの「両方可視」を本物の秘匿（自分の手のみ可視、相手は reveal まで気配のみ）へ。観戦配信では秘匿境界の設計。
- **合法手提示の美意識調整**: 水墨に最も馴染む提示表現を、実地で見ながら詰める。
- **CPU 対戦**: ルールが固まってから。不完全情報ゲームの思考機械（CFR/ナッシュ近似等）。
- 各殻は独立に足せる。本段階は「一人で両者を指せて、合法手が水墨の静けさの中で提示され、棋譜が一本に積まれる」ことに集中する。
