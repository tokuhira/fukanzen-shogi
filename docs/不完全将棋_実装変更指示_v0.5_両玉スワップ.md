# 不完全将棋 実装変更指示 v0.5 — 両玉スワップの相討ち

> 対象実行者: Claude Code
> 前提: v0.4 まで実装済み（戦国無双特則がスワップ 4.4 と同一マス相討ち 4.3 をカバー）。
> 関連文書: `不完全将棋_ルール仕様_v0.5.md`（正本）。
> 性格: 既存実装への一点追加。エンジンの衝突解決のうち、戦国無双特則が「両当事者がともに玉」のケースを未定義のまま残していた穴を塞ぐ。合法性判定は変更しない。

---

## 0. 背景

TUI 検証卓でのプレイ中に発見された現象。両玉が間に2マスを空けて対峙した状態から、両者が同時に相手の方へ一歩進むと、各々の移動先は着手開始時点では相手玉の利きの外にあり双方合法だが、移動後に**両玉が隣接する**（伝統的将棋では手番交互ゆえ起こり得ないが、手番の消滅した本ゲームでは同時に踏み込めるため成立する）。

隣接した次のターン、両玉が互いに相手玉のマスへ進むと**両玉のスワップ**になる。v0.4 までの戦国無双特則は「当事者の**一方**が玉」しか想定しておらず、「**両方**が玉」のスワップが未定義だった。「玉が勝つ」を両者へ適用すると矛盾する。

---

## 1. 変更の本体（仕様書 v0.5 の 4.7 追加条項）

> **スワップ（4.4）の両当事者がともに玉のとき、双方の戦国無双が拮抗して相殺し、通常の相討ち（4.4）に戻る。両玉が取られ、引き分けとなる。**

すなわち、戦国無双特則の「玉が当事者なら玉が勝つ」を、**「ただし両当事者がともに玉なら、相殺して通常の相討ち（両玉とも取られる）」**で上書きする。結果は両玉の死＝引き分け（5.4）。

### 1.1 なぜ「両者裸」しか起こらないか（合法性が自動的に担保）

両玉スワップが合法に成立するのは、両玉とも後ろ盾を持たない（互いの移動先＝相手玉のマスが着手開始時点で安全な）ときに限られる。片方の玉に後ろ盾があれば、もう一方の玉はその守られたマスへ侵入できず（玉の侵入禁止）、玉同士のスワップはそもそも `legal_actions` に現れない（後ろ盾を持つ側の玉だけが、裸の相手玉を一方的に取りに行ける＝通常の戦国無双）。**この絞り込みは既存の合法性判定が担保しており、resolve 側で後ろ盾を再判定する必要はない。**

---

## 2. 実装箇所

衝突解決（`engine/src/resolve.rs` 等）のスワップ分岐、戦国無双の「一方が玉」判定の箇所。

- 現状: スワップで「当事者の一方が玉」なら玉が勝つ。
- 追加: その判定の前に「**両当事者がともに玉**」を分岐し、その場合は戦国無双を適用せず**通常の相討ち**（両玉とも取られる）として解決する。両玉が取られるので、終了判定は**引き分け**（両玉同時の死、5.4）。
- 同一マスの相討ち（4.3）側は、両玉が共通マスへ進む形が**そもそも合法に発生しない**（共通マスは双方の玉に隣接＝相手玉に利かれており、いずれの玉も侵入できない）ため、4.3 分岐には両玉ケースの追加は不要。ただし防御的に、4.3 で両当事者が玉なら相討ち（引き分け）として扱う実装にしておいても害はない（到達しないはずの分岐）。

> 玉は持ち駒にならない（既存方針）。両玉相討ちは「両玉を持ち駒に加える」のではなく、両者の玉が取られた＝引き分けの終了フラグとして扱う。

## 3. テストの変更

### 3.1 追加するテスト

- **両玉スワップ → 相討ち引き分け**: 両玉が隣接し（後ろ盾なし）、互いに相手玉のマスへ進むスワップで、**両玉が取られ、引き分け**が宣言されることを確認。
- **両玉隣接が合法に成立する**: 両玉が間に2マスを空けて対峙した局面から、両者が同時に相手方向へ一歩進む着手がともに合法手として生成され（着手開始時点では互いの移動先が相手玉の利きの外）、解決後に両玉が隣接することを確認。
- **後ろ盾ありなら玉スワップは非合法（非対称になる）**: 片方の玉に後ろ盾がある隣接局面で、裸側の玉が後ろ盾持ちの玉のマスへ進む手は `legal_actions` に現れず（侵入禁止）、後ろ盾を持つ側の玉だけが裸の相手玉を取りに行ける（通常の戦国無双で一方的に取れる）ことを確認。

### 3.2 退行防止

- 「一方が玉・相手が玉でない」スワップ／同一マス相討ち（v0.3・v0.4 の戦国無双）は従来どおり**玉が勝つ**こと。両玉ケースの追加が、片方だけ玉のケースを壊していないことを確認。
- v0.4 までの全テスト（取り合いの裏目・合駒貫き・逃げ先への打ち込みを玉が制する等）が引き続き通る。

### 3.3 見直し

- v0.4 までに「玉同士の相討ちは発生し得ない」前提で書かれたテスト・コメントがあれば、**訂正する**。両玉隣接スワップで実際に発生する。

---

## 4. やらないこと（非ゴール）

- **両玉の隣接を禁止しないこと。** v0.5 は「出口で裁く」方針を採る。両玉が隣接し得ることを正式に認め、隣接後のスワップを相討ち引き分けで裁定する。「入口で塞ぐ」案（玉は移動後に敵玉へ隣接する手を指せない）は採用しない（将来の検討余地としてのみ残す）。
- 合法性判定（玉の侵入禁止）は一切変更しない。両玉スワップが「両者裸のときだけ合法」という絞り込みは既存ロジックが担保する。
- 千日手・連続王手・打ち歩詰め・入玉宣言などの未確定事項、後続段階には触れない。

## 6. 検証後の所見記録（推奨）

v0.5 は終盤に「一騎打ち」という新しい様式を導入する。TUI 検証卓で終盤を指してみて、次を所見として記録すること（修正はせず所見に留める）:

- 両玉隣接からの一騎打ちが、豊かな終盤の駆け引きとして機能するか。
- 相討ち引き分けが過度に頻発したり、間合いの取り合いが膠着を招いたりしないか。
- 望ましくなければ「入口で塞ぐ」案（玉の敵玉隣接禁止）への切り戻しを検討（人間の判断）。
